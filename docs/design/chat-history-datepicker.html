Assistant chat feed & transaction date picker draft · rev 1

Assistant tab becomes a daily conversation feed: your words sit on the right, the resulting transaction record on the left. Manually-added entries appear as a compact record card (a mini version of the Transactions-list row) on the right. The feed shows today only and resets each day. The avatar is adaptive — it fills the screen when the feed is empty (start of day) and collapses to a compact header as messages accumulate, so the feed gets the room when it has content (no setting needed). Transaction edit swaps the free-text date for a native date picker shown as dd-MM-yyyy.

Avatar is extensible by kind. Today there's one kind — Blob (Xavier), the default — with colour variants. The model leaves room for future kinds (Character, Animated — Lottie/Rive/AI art) without a rewrite: each kind is a renderer that must support the 5 states (idle/listening/thinking/happy/confused). Domain (avatar.ts) holds kinds + variants as data; a component registry maps kind → renderer; AssistantAvatar stays the single swap point. Existing saved looks map to kind: 'blob' — no migration.
How the feed maps to data (no risk to financials). The feed is rendered from today's transactions (createdAt within the current day), newest at the bottom. Transactions are never deleted — they remain on the Transactions tab and in net-worth math. "Resets daily" only means the conversation view shows the current day; at midnight it simply starts empty and fills from that day's activity. AI entries store the user's original utterance (a new sourceText column) to render the right-side bubble; manual entries have no utterance, so they show as the compact record card on the right.
Date picker. Replaces the free-text YYYY-MM-DD input in the transaction sheet with a tappable field that opens @react-native-community/datetimepicker (native, iOS spinner / Android dialog). Display + parse format is dd-MM-yyyy. Accounts are unchanged (no date concept). This is a native module: adding it needs a dev-client rebuild (eas build --local --profile development), not just a JS reload.
