Making the app scale properly across devices

The app hard-codes point sizes tuned for a ~390 pt canvas and ignores iOS Dynamic Type. On a 430 pt iPhone 15 Pro Max the same text fills a smaller fraction of the screen and the centered hero leaves big empty bands — so it reads as "zoomed out." The /account question, chips and step line inherit small roles, so that step feels especially cramped. Below: the root causes, a responsive scale model, and how the two affected screens look before vs. after.

The scale model

Every size is derived from a role, then multiplied by two factors: device width and the user's Dynamic Type setting. Spacing and touch targets scale with width too, so a bigger phone breathes instead of leaving gaps.

Formula
size = base × widthFactor × dynamicType
widthFactor = clamp(0.94, screenWidth / 390, 1.12) — the width factor and Dynamic Type multiplier are the only two knobs; everything else falls out of the role table.

What to change in code

1 · A useScaledType() hook
Replace fixed typography.* reads with a hook returning base × widthFactor × PixelRatio.getFontScale(). widthFactor from useWindowDimensions().
2 · Add the prompt role
Add prompt: 22 to the type scale and use it for the assistant question in index.tsx instead of text-base.
3 · Fix the touch targets
Subtype & quick-action chips: set minHeight 44 (48 on Pro Max) instead of py-2. Same for the composer buttons.
4 · Let the hero breathe
Drop the fixed minHeight 340; distribute space with flex and scale the avatar with width so tall screens fill instead of centering a small cluster.
